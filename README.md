# Вместе

Учебный веб-проект по теме "Искусственный интеллект в психологии" (вариант №3 - полноценное приложение/вебсайт).

## Концепция

`Вместе` - это прототип цифрового помощника, который:
- мягко устанавливает контакт;
- проводит обязательный минимум вопросов безопасности;
- предлагает короткие техники стабилизации;
- выдает ресурсы помощи (102/103 + региональные контакты-заглушки);
- формирует персональный план на 24 часа.

## Реализовано в интерфейсе

- Кнопка `Быстро закрыть` (переключение на нейтральную страницу).
- Режим `кто-то рядом` (короткие ответы).
- `Автоочистка истории` каждые 30 секунд.
- Сценарный диалог по этапам: город -> возраст -> 4 риска -> ветвление -> стабилизация -> план.
- Раздел `План на 24 часа`, который заполняется динамически.

## Технологии

- HTML
- CSS
- JavaScript (Vanilla JS)
- Node.js + Express + официальный SDK [**@google/genai**](https://ai.google.dev/gemini-api/docs/quickstart?hl=ru) (Gemini API)

## Запуск

1. Скопировать `.env.example` в `.env`.
2. Получить бесплатный ключ в [Google AI Studio](https://aistudio.google.com/apikey) и вставить в `GEMINI_API_KEY`.
3. При необходимости сменить модель в `GEMINI_MODEL` (см. [документацию и список моделей](https://ai.google.dev/gemini-api/docs/quickstart?hl=ru)). Устаревшее имя **`gemini-pro` не использовать**.
4. Рекомендуется **Node.js 20+** (требование SDK `@google/genai`).
5. Установить зависимости: `npm install`.
6. Запустить: `npm run dev`.
7. Открыть **именно** `http://localhost:3000` (не открывайте `index.html` двойным кликом и не используйте Live Server на другом порту без запущенного `npm run dev`).
8. Проверить работу:
   - `Быстро закрыть`,
   - режим коротких ответов,
   - автоочистка истории,
   - формирование плана на 24 часа,
   - статус `AI: online` в блоке чата.

## Важно

Это учебный прототип, не медицинская система. Приложение не ставит диагноз и не заменяет очную помощь.

## Если в чате «AI: offline»

1. В терминале в папке проекта должно быть запущено `npm run dev` (окно не закрывать).
2. Браузер: откройте **`http://127.0.0.1:3000`** (или `http://localhost:3000` с тем же портом).
3. В `.env` должен быть **`GEMINI_API_KEY`** без лишних кавычек и пробелов; после правки `.env` перезапустите сервер.
4. Если видите сообщение с текстом ошибки от API — чаще всего это неверный/отозванный ключ, лимит бесплатного тарифа или недоступная в вашем регионе модель (попробуйте сменить `GEMINI_MODEL` на `gemini-1.5-flash`).
